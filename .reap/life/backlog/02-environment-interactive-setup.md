## 현재 동작
- `reap init` 시 `.reap/environment/` 빈 폴더만 생성
- objective 단계에서 environment가 비어있으면 안내 메시지만 출력
- 사용자가 직접 파일을 넣어야 함

## 개선 방향
- `/reap.objective`의 Environment Scan 단계에서, 비어있을 경우:
  1. 먼저 Environment 개념을 브리핑:
     "Environment는 이 프로젝트 외부에서 주어지는 정보를 기록하는 곳입니다.
      프로젝트가 직접 통제할 수 없지만 개발에 영향을 주는 것들 — 연계 시스템,
      인프라 환경, 조직의 규범이나 가이드라인, 외부 API 스펙 등이 해당됩니다.
      Genome(설계와 지식)과 달리, Environment는 외부 세계의 제약과 컨텍스트입니다."
  2. 브리핑 후 대화형으로 정보 수집
- 질문 예시 (시스템 외부에서 주어지는 정보에 집중):
  - "연계 시스템이 있나요? (다른 서비스, 외부 API, 레거시 시스템 등)"
  - "인프라/배포 환경은 어떻게 되나요? (클라우드, 온프레미스, 컨테이너 등)"
  - "조직에서 관리하는 규범이나 가이드라인이 있나요? (회사 내규, 코딩 표준, 보안 정책, 법규 준수 요건 등)"
  - "외부에서 제공받는 참고 문서가 있나요? (API 문서, 연계 시스템 스펙, 인프라 구성도 등)"
- 수집한 정보를 `.reap/environment/`에 구조화된 마크다운으로 자동 저장
- 이미 environment 파일이 있으면 추가 수집 여부만 확인

## 기대 결과
- 첫 generation에서 environment가 자연스럽게 채워짐
- 사용자가 "뭘 넣어야 하지?" 고민할 필요 없음
