# Auto Issue Report 기능 추가

## 개요
REAP에서 malfunction 발생 시 에이전트가 GitHub Issue를 자동 등록하는 기능.

## 설치 시 동작
1. `reap init` / `reap update` 시 `gh` CLI 존재 확인
2. `gh` 있으면 → config.yml에 `autoIssueReport: true` 기본 활성화
3. `gh` 없으면 → GitHub CLI 설치 권장 메시지 + `autoIssueReport: false`

## config.yml
```yaml
autoIssueReport: true
issueRepo: c-d-cc/reap
```

## Malfunction 감지 기준
- current.yml 파싱 실패 / 불일치
- reap.next stage 전환 시 예상과 다른 상태
- compression 실행 중 에러
- hook 실행 실패
- reap fix에서 문제 발견
- 커맨드 Gate 조건과 실제 상태 불일치

## Issue 형식
```markdown
## Bug Report (Auto-generated by REAP)

**REAP Version**: v{version}
**Node Version**: v{nodeVersion}
**OS**: {os}
**Agent**: {agentName}

### Context
- **Generation**: {genId} (stage: {stage})
- **Command**: {command}
- **Genome Hash**: {genomeHash}

### Error
{에이전트가 감지한 malfunction 설명}

### Expected Behavior
{정상 동작 설명}

### Actual Behavior
{실제 발생한 동작}

### Steps to Reproduce
1. {step}

### Logs
{관련 에러 로그 — sanitized}

### Config (sanitized)
{config — 민감정보 제거}

---
*Auto-reported by REAP agent. Repository: c-d-cc/reap*
```

## 개인정보 보호 — 이중 검사 (CRITICAL)

### Rule: 절대로 사용자 개인정보, 민감정보, 프로젝트 내부 내용을 issue에 포함하지 않는다.

### 1차 검사: 템플릿 포맷 전 (Pre-format Instruction)
에이전트에게 issue 내용을 작성하기 전에 다음 지시를 주입:
```
BEFORE writing the issue report:
- Do NOT include: file contents, source code, environment variables, API keys,
  tokens, passwords, usernames, email addresses, IP addresses, domain names,
  database URLs, project-specific business logic, or any proprietary information.
- ONLY include: REAP version, Node version, OS, generation ID, stage name,
  command name, genome hash, generic error messages, and REAP config (with
  sensitive fields masked as ***).
- When in doubt, OMIT the information rather than include it.
```

### 2차 검사: 포맷 후 (Post-format Sanitization)
생성된 issue 본문에 대해 다음 패턴을 검사 + 마스킹:
- 이메일 패턴: `*@*.* → ***@***`
- API key/token 패턴: `sk-*`, `ghp_*`, `Bearer *` 등 → `[REDACTED]`
- URL에 credential 포함: `https://user:pass@` → `https://***:***@`
- 환경변수 값: `KEY=value` → `KEY=[REDACTED]`
- 파일 경로 중 사용자명: `/Users/{username}/` → `/Users/***/`
- IP 주소: `\d+\.\d+\.\d+\.\d+` → `[REDACTED_IP]`
- 소스코드 블록: 실제 프로젝트 코드가 포함되면 제거
- 최종적으로 유저에게 보여주고 confirm 받은 후에만 등록

### 유저 확인 흐름
1. Malfunction 감지
2. "문제가 감지되었습니다. GitHub Issue로 보고할까요? 이 리포트는 REAP 프로젝트의 품질 개선에 큰 도움이 됩니다. (yes/no)"
3. yes → issue 본문 생성 (1차 검사 적용)
4. 생성된 본문에 2차 검사 (자동 마스킹)
5. 마스킹된 최종 본문을 유저에게 표시: "아래 내용으로 issue를 등록합니다. 민감한 정보가 포함되어 있지 않은지 확인해주세요."
6. 유저 최종 confirm → `gh issue create`
7. 등록 완료 후: "Issue가 등록되었습니다. 리포트에 감사드립니다! 여러분의 피드백이 REAP을 더 나은 도구로 만듭니다."

## 수정 대상
- `src/cli/commands/init.ts` — gh 존재 확인 + config 설정
- `src/templates/commands/reap.*.md` — malfunction 감지 시 issue report 제안 지시 추가
- 새 커맨드 또는 유틸: issue 포맷 + sanitization 로직
- reap-guide.md — autoIssueReport 설명 추가
